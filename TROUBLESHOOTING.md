## 2. 적용한 기술 (Technologies & Concepts)

### 2.1 로봇 시각화
- **`Visualizer2D`** (Mobile Robotics Simulation Toolbox): 2D 모바일 로봇 시뮬레이션 화면.
- **Pose 표현 `[x; y; θ]`**: 위치(x, y) + 자세(헤딩 θ).
- **헤딩 계산**
  - 사각형: 변의 방향 벡터로 `θ = atan2(dy, dx)` → 항상 진행 방향을 바라봄.
  - 원: 접선이 위치각보다 90° 앞서므로 `θ = φ + π/2`.
- **선형 보간(linear interpolation)**: 두 점 사이를 `start + ratio*(end-start)`로 잘게 나눠 이동.

### 2.2 애니메이션 / GIF 저장
- **GIF 녹화**: `getframe` → `rgb2ind`(256색) → `imwrite`.
  첫 프레임은 `'LoopCount', inf`, 이후 `'WriteMode', 'append'`로 이어붙임.
- **재생 속도**: `'DelayTime'` (프레임 간 간격). `pause()`(실행 중 보기 속도)와 구분됨.
- **렌더링 최적화(프레임 스킵)**: 물리 계산은 매 스텝, **화면 갱신·GIF 저장은 N스텝마다**(`plotEvery`).
  `pause` 대신 `drawnow limitrate` 사용 → 보기 속도 대폭 향상, GIF 용량 감소.
- **축 고정**: `axis([...])`로 시야를 고정해 프레임마다 흔들리는 현상 제거.

### 2.3 이미지 → 지도 (Occupancy Grid)
- **`imread` → `rgb2gray` → 임계값(threshold)** : `grayimage < 128`로 흑백(이진) 분류.
  - 흰색 = 빈 공간(로봇 통행), 검은색 = 장애물.
- **`binaryOccupancyMap`**: 흑백 행렬을 점유 격자 지도 객체로 변환.
- **`imresize`로 축소**: 격자 수↓(성능), 로봇 가시성↑.
  좌표는 축소 비율에 **정비례** → waypoint를 `× (scale/0.1)`로 자동 환산.
- **모폴로지 정리(`bwareaopen`)**: 글자·주차선 등이 만든 잡티(작은 점)를 제거해 길 연결성 확보.

### 2.4 자율주행 파이프라인 (보너스)
- **안전여유(Configuration Space)**: `copy(map)` 후 `inflate(map, r)`로 장애물을 로봇 반경만큼 부풀려
  계획용 "안전 지도" 생성 (벽에 붙는 경로 방지).
- **전역 경로계획(Global Planning)**: `mobileRobotPRM` + `findpath`로 건물을 피해 경유점들을 잇는 경로 생성.
  여러 waypoint를 구간별로 계획해 이어붙임(route stitching).
- **경로 보간(densification)**: PRM 경로의 듬성한 점 사이를 ~2m 간격으로 채워 추종 정밀도↑.
- **경로추종(Path Following)**: `controllerPurePursuit`
  (`LookaheadDistance`, `DesiredLinearVelocity`, `MaxAngularVelocity`).
- **로봇 모델**: `differentialDriveKinematics` + `derivative`로 차동구동 운동학 시뮬레이션
  (오일러 적분: `pose = pose + vel*dt`).
- **시작 방향 정렬**: 첫 경로 방향으로 `θ₀`를 맞춰 출발 직후 회전 루프 방지.
- **진단 도구**: `checkOccupancy`(좌표 점유 확인), `show(map)`·`show(planner)`(연결성 시각화).

## 3. 트러블슈팅 기록 (Problem → Cause → Solution)

### 3.1 `Visualizer2D` 사용 시 경고
- **증상**: `Warning: System Object 'Visualizer2D' ... mixin 'CustomIcon' ... no longer be supported`
- **원인**: 툴박스 내부 코드가 구버전 방식으로 작성됨 (내 코드 문제 아님).
- **해결**: 경고(Warning)일 뿐 실행에 지장 없음 → 무시. (에러/경고 구분 학습)

### 3.2 섹션(`%%`) 단위 실행
- **증상**: `Ctrl+Enter`로 한 단계씩만 실행됨.
- **원인**: `%%`는 새 섹션 시작, `%`는 단순 주석. 단계마다 `%%`를 써서 잘게 쪼개짐.
- **해결**: 변수를 주고받는 다중 섹션 작업(5번·보너스)은 **전체 선택 후 F9**로 실행.

### 3.3 그래프가 매 프레임 흔들림
- **원인**: `Visualizer2D`가 프레임마다 축 범위를 자동 재계산(auto-scale).
- **해결**: `viz(...)` 다음에 `axis([...])`로 축 범위 고정.

### 3.4 결과물을 영상/GIF로 저장
- **`.fig`** = MATLAB 전용 편집 파일(깃허브 미리보기 X) → **`.png`/`.gif`**로 저장해야 함.
- **영상은 끝나고 저장 불가** → 루프 안에서 프레임을 모아야 함(`getframe`+`imwrite`).
- GIF 재생이 너무 빠름 → `DelayTime` 0.02 → 0.05.
- 화면 보기가 느림 → 프레임 스킵(`plotEvery`)으로 해결.

### 3.5 MATLAB에서 GIF 더블클릭 시 Import Wizard 등장
- **원인**: MATLAB이 GIF를 "데이터(변수)"로 가져오려 함.
- **해결**: Cancel. GIF 재생은 `winopen(pwd)`로 폴더 열어 브라우저로 보기.

### 3.6 `imread` 파일 없음 에러 / `bKGD` PNG 경고
- `File ... does not exist`: 이미지가 작업 폴더(`pwd`)에 없음 → 같은 폴더에 저장.
- `PNG library warning: bKGD: invalid`: 배경색 메타데이터가 규격에 약간 안 맞음 → 안 쓰는 정보라 무시 가능.

### 3.7 지도 축소 후 길이 끊김 ★
- **증상**: 원본엔 멀쩡한 길이 PRM에서 끊겨 경로 탐색 실패.
- **원인**: `imresize(bwimage, 0.1)`로 1/10 축소 시, 가는 길이 **1칸 미만**으로 줄어 사라짐.
- **해결**: 축소 비율 0.1 → 0.35로 완화. 좌표는 `× (scale/0.1)`로 자동 환산해 재선택 불필요.

### 3.8 PRM "경로를 못 찾음" (구간 2→3, 5→6) ★
- **진단 1**: `checkOccupancy(map, waypoints)` → 모든 경유점은 빈 칸(0). 위치 문제 아님.
- **진단 2**: `show(map)` 시각화 → 흰 길 속에 **검은 잡티(글자·주차선)** 가 길을 잘게 끊고 있었음.
- **해결**:
  1. 모폴로지 정리(`bwareaopen`)로 잡티 제거 → 길 연결성 회복.
  2. 그래도 막힌 5→6은 **waypoint 6 제거**(요구사항이 "5개 이상"이라 5개로 충족).

### 3.9 `bwareaopen` 함수 없음
- **증상**: `Undefined function 'bwareaopen' ...`
- **원인**: **Image Processing Toolbox 미설치**.
- **해결**: Add-Ons에서 해당 툴박스 설치 (정석). + 이진맵은 `imresize(..., 'nearest')`가 적합.
- **교훈**: 워크어라운드(직접 구현)보다, 가능하면 **필요한 툴박스를 설치**하는 것이 깔끔하고 재사용성이 높다.

### 3.10 로봇이 경로를 무시하고 벽을 뚫음 ★★ (핵심)
- **진단**: `show(map)` 위에 계획 경로(`route`)를 그려보니 **초록 경로는 벽을 완벽히 회피**.
  → 즉 **계획(PRM)은 정상, 추종(Pure Pursuit)이 문제**임을 분리해서 확인.
- **원인 1 (추종 튜닝)**: `LookaheadDistance`가 커서 코너를 크게 잘라먹고 오버슈트 → 큰 루프.
  → 10→5, 속도 6→3, `MaxAngularVelocity` 3→4로 조정 + 경로 보간 + 시작 방향 정렬.
- **원인 2 (구조적 한계)**: `controllerPurePursuit`은 **전진 전용(후진 불가)**.
  급커브(U턴)에서 방향 전환을 못 해 작게 돌고, 그 과정에서 경로를 벗어나 벽 침범.

### 3.11 "벽 회피"와 "경로추종"은 충돌인가? → 아니오, **구조적 빈틈**
- 벽 회피는 **계획 단계(PRM+inflate)** 에만 존재 → "안전한 경로"만 보장.
- `controllerPurePursuit`은 **맵·장애물을 전혀 모름**(충돌검사 없음) → 경로를 벗어나면 막을 장치가 없음.
- 즉 두 로직이 싸운 게 아니라, **주행 중 충돌을 감시하는 층(지역 회피)이 부재**한 것.

## 4. 학습하며 가졌던 의문점 (Q & A)

- **Q. 경고(Warning)와 에러(Error) 차이?** → 경고는 실행되지만 주의, 에러는 실행 중단.
- **Q. 과제마다 파일을 따로 만들어야 하나?** → `%%` 섹션으로 한 파일에 나눠 담을 수 있음.
- **Q. GIF는 프레임을 이어붙인 것뿐인데 괜찮나?** → 맞음. GIF=프레임 통째 저장(256색·무압축에 가까움), MP4=차분 압축. 단순 시뮬엔 GIF가 적합(깃허브 자동재생).
- **Q. 점이 화면 안에 있어서 축을 고정할 수 있는 건가?** → 그렇다. 로봇이 머무는 범위를 알기에 그 범위로 축을 고정 가능.
- **Q. Task 3과 4는 같은 건가?** → 아니다. 3=직접 만든 맵, 4=실제 지도 캡쳐 활용. 코드는 유사하나 "맵의 출처"가 다름.
- **Q. "자신만의 지도"는 꼭 손으로 그려야 하나?** → 아니다. 커스텀 맵이면 됨(캡쳐·트레이스 가능).
- **Q. 자율주행하려면 꼭 흑백이어야 하나? 유색이면 인식 못 하나?** → 로봇은 색이 아니라 **막힘/뚫림**만 봄. 최종적으로 이진 변환은 필수이며, 유색은 변환 정확도가 떨어질 뿐.
- **Q. 왜 맵을 축소하나? 큰 숫자가 불편해서?** → 진짜 이유는 **성능(격자 수)** 과 **로봇 가시성**. 좌표 편의는 부수 효과.
- **Q. 회피하려면 Lidar가 필요한가?** → **전역 회피**(맵 기반 계획)는 불필요, **지역 회피**(실시간 감지)는 Lidar 필요.
- **Q. Pure Pursuit이 검은 건물을 자동으로 피하나?** → 아니다. 주어진 경로만 따라감(맵 모름).
- **Q. waypoint만 찍는 건 쓸모없지 않나?** → 아니다. waypoint = 주행의 입력(경로)이며 그대로 재사용됨.
- **Q. 화면 속도(`plotEvery`)를 바꾸면 경로도 바뀌나?** → 아니다. 물리·제어는 그대로, 보여주는 빈도만 변함.

## 5. 한계점 & 향후 개선 (HW#4와의 비교)

이 보너스 구현은 **자율주행 파이프라인을 이해·연결한 데모**이지만,
**완성된 안전 주행은 아니다.** (실제로 일부 벽을 침범함)

### 한계점
1. **주행 중 충돌 검사 부재** — 벽 회피가 계획에만 있고 실행엔 없음.
2. **전진 전용(후진 불가)** — U턴에서 루프 발생, 경로 이탈 가능.
3. **실시간 감지 없음** — Lidar 부재로 지역 회피 불가.
4. **맵이 근사치** — 축소·잡티 제거로 세부가 손실됨.
5. **튜닝 의존적** — 파라미터에 민감, 일반화 어려움.
6. **이상적 시뮬레이션** — 완벽한 위치 추정 가정(센서 오차·슬립 무시).

### HW1(보너스) ↔ HW4 비교
| 항목 | HW1 보너스 | HW4 |
|------|-----------|-----|
| 전역 경로계획 (충돌 없는 경로) | PRM ✅ | PRM ✅ |
| 경로추종 | Pure Pursuit ✅ | Pure Pursuit ✅ |
| **주행 중 실시간 보정** | ❌ → 경로 이탈 시 벽 침범 | ✅ Lidar + VFH |

> **핵심**: 두 과제 모두 "충돌 없는 경로"는 만들었다 — 차이는 **"올바른 계획 ≠ 안전한 실행"**,

# HW#3 — Lidar Line Feature Extraction (개발 기록)

> **문제**: 라이다가 0°~80°를 5° 간격으로 측정한 17개의 거리값(θ, ρ)이 주어진다.
> 잡음이 있는 이 점들에 가장 잘 맞는 **직선(벽)** 을 찾아 극좌표 normal form (r, α)로 표현하는 것.
> 목표값: α ≈ 37.36°, R ≈ 0.4
> 파일: `AutoDrive_HW3_LineFeatureExtraction.m`

## 적용 기술

- 극좌표 → 직교좌표 변환 (`x = ρcosθ`, `y = ρsinθ`)
- **일반 최소제곱** (`polyfit`, y방향 오차 최소화)
- **Total Least Squares** 닫힌 해 (직선까지의 수직거리 오차 최소화) — 강의자료 공식
- 직선의 극좌표 normal form `(r, α)` 표현 및 시각화

## 트러블슈팅 / 의문점

### 1. 왜 정답 37.36°이 아니라 38.85°가 나오나?
- **원인**: `polyfit`은 **y방향 거리**를 최소화(일반 최소제곱)하고, 강의 공식은 **직선까지의 수직거리**를 최소화(Total Least Squares)한다. 오차를 재는 기준이 달라 약 1.5° 차이가 난다.
- **이해/해결**: 라인 피팅의 정석은 수직거리 최소화(방법 B) → **37.96°** 로 정답과 일치. `polyfit`(38.85°)도 교수님이 "유사답 허용"이라 통과.

### 2. r 변환식 버그
- **기존**: `R = b/sqrt(a^2 + 1 + b^2)` → `+ b^2` 가 잘못 들어가 0.3715 산출.
- **수정**: `R = |b|/sqrt(a^2 + 1)` (정식 변환식, 0.4002). 반올림하면 둘 다 0.4라 결과는 통과했지만 식 자체를 바로잡음.

### 3. 강의 공식의 부호(quadrant) 문제
- 닫힌 해를 `0.5*atan2(num, den)` 로 그대로 쓰면 **-52.04°**(= 정답 - 90°, 즉 직선의 *방향* 각도)가 나온다.
- **원인**: `atan2` 인자 부호. 우리가 원하는 건 직선의 **법선(normal)** 각도 α.
- **해결**: `0.5*atan2(-num, -den)` 로 부호를 맞추면 **37.96°** (정답과 일치).

## 결론

- **방법 B(TLS)** 가 강의자료 정답(37.36°, 0.4)과 일치하는 **정석 풀이**.
- **방법 A(`polyfit`)** 는 간단하지만 y방향 오차 기준이라 각도가 약간 다르다. 두 방법 모두 r ≈ 0.4.
- 핵심 학습: **"어떤 오차를 최소화하느냐"(수직 vs 수평거리)에 따라 결과가 달라진다**는 점.

# HW#4 — 모바일 로봇 자율주행 (개발 기록)

> **문제**: 집 평면도 맵 위에서 차동구동 로봇이 Lidar로 주변을 감지하며,
> 시작점→목표점까지 벽·가구·장애물을 피해 자율주행.
> 파일: `AutoDrive_HW4_Navigation.m`

## 적용 기술 (3층 구조)

- **전역 경로계획(Global)**: `mobileRobotPRM` + `findpath` — 맵 전체를 보고 충돌 없는 경로 생성
- **지역 회피(Local)**: `controllerVFH` — Lidar로 주변을 보고 실시간 회피
- **모션 제어**: `controllerPurePursuit` + `differentialDriveKinematics`
- 센서: `LidarSensor`, `ObjectDetector`(장애물 6개)
- 맵: `binaryOccupancyMap`, `imdilate`(선 보강), `imresize`(축소), `inflate`(안전여유)

## 트러블슈팅 (실제로 겪은 순서)

### 1. 맵이 고해상도 → 화면이 멈춘 듯 느림
- **원인**: 1000×678 ≈ 68만 칸. 매 스텝 Lidar가 거대한 격자에 빔을 쏘고 PRM도 무거워 사실상 정지.
- **해결**: `imresize`로 축소(셀 수↓). `resolution`을 함께 줄여 **실제 맵 크기(10×6.8m)는 유지** → 좌표 그대로 사용.

### 2. 선을 너무 두껍게 → 문(통로)이 막혀 경로 탐색 실패
- **원인**: 축소 시 가구 선 소실을 막으려 과하게 부풀림(`disk 3` + 임계값 0.25) → 벽이 두꺼워져 문이 닫힘 → `findpath` 실패.
- **해결**: `disk 2` + 재이진화 임계값 **0.5**(벽 과팽창 방지) + 축소 0.35 + `robotRadius`↓.

### 3. goal 위치가 점유됨 (Invalid 'goal' location)
- **원인**: 시작/목표가 벽·가구(또는 inflate 후 점유) 위에 있음.
- **해결**: `show(mapInflated)` 위에서 `ginput`으로 **확실히 빈(흰) 곳** 2점 선택.

### 4. viz가 아무 그림도 안 그림
- **원인**: 앞서 Ctrl+C로 멈출 때 `Visualizer2D` 객체가 **잠긴 채 손상**됨.
- **진단**: 새 `Visualizer2D` 와 `show(map)` 는 정상 → 기존 viz만 문제.
- **해결**: 주행 단계에서 **viz를 새로 생성**해 사용.

### 5. 출발 못 하고 제자리 회전 (VFH가 NaN 반환)
- **원인**: 닫힌 집이라 Lidar가 5m 안 **사방에서 벽**을 봄. VFH `DistanceLimits`를 5m까지 두니 "모든 방향 막힘"으로 판단.
- **해결**: `DistanceLimits = [0.05, 1.5]` → **가까운 장애물만** 고려.

### 6. 소파 근처에서 갇혀 맴돌이 (전역 vs 지역 충돌) ★ 핵심
- **원인**: Pure Pursuit 전방주시점(0.5m)이 **소파 너머**를 가리킴 → VFH는 그 방향을 막음 → 둘이 충돌해 제자리 맴돔.
- **해결**: 경로를 **촘촘히 보간(0.15m)** + 전방주시 **0.3m**로 단축 → 로봇이 (소파를 피한) PRM 경로에 밀착 → VFH와 충돌 해소.

### (실패한 시도) 시작 맴돌이 줄이려 속도를 조향각에 연동
- `v = 0.4*cos(steerDir)` 적용 → 크게 틀어야 하는 출발 시 `v≈0` → **앞으로 안 가고 회전만 하는 교착**.
- → 되돌림. (시작의 작은 호는 전진 전용 로봇의 자연스러운 전환 현상으로 수용)

## 결론 / 교훈

- **자율주행은 3층(전역계획 → 지역회피 → 모션제어)이 서로 조화**되어야 한다.
  - 전역계획만 좋아도(HW1) 실행이 못 따라가면 벽을 뚫고,
  - 지역회피가 과민하면(먼 벽까지 고려) 출발조차 못 하며,
  - 둘이 충돌하면(전방주시 과대) 제자리 맴돈다.
- 성능(맵 해상도)과 정확도(선 두께·통로)는 **trade-off**이며, 둘을 함께 맞춰야 한다.
- HW1의 한계(충돌 검사 부재 → 벽 침범)를 HW4에서 **Lidar+VFH 지역 회피**로 해결 → "before→after" 완성.
